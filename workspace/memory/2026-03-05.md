## Leçon : Handoff cross-canal (webchat ↔ Telegram)
- **Problème identifié :** Quand Sylvain demande de poursuivre une conversation d'un canal à un autre (ex: webchat → Telegram), le contexte conversationnel ne suit pas automatiquement. Chaque canal a sa propre session avec son propre historique.
- **Cause :** L'agent source (webchat) a envoyé un message sur Telegram mais n'a pas écrit de note de contexte dans `memory/` pour que l'agent destination (Telegram) puisse reprendre le fil.
- **Solution obligatoire pour tout handoff cross-canal :**
  1. Écrire un fichier `memory/handoff-YYYY-MM-DD-HHMMSS.md` avec : résumé de la conversation, ce que l'utilisateur veut faire, prochaines étapes
  2. Le message envoyé sur le canal destination doit référencer ce contexte
  3. À la réception d'un nouveau message sur n'importe quel canal : vérifier s'il existe un handoff récent dans `memory/`
- **Règle :** Avant de répondre à un premier message d'une session, TOUJOURS faire `memory_search` pour du contexte récent cross-canal.
